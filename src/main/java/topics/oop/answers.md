### 1. Назовите основные принципы ООП. Расскажите подробно о каждом.
- **Инкапсуляция** – скрытие данных внутри объекта.
- **Наследование** – создание новых классов на основе существующих.
- **Полиморфизм** – способность методов работать с разными типами данных.
- **Абстракция** – выделение только значимых характеристик объекта.

### 6. Расскажите про модификаторы доступа, к чему они применяются.
Модификаторы — это специальные слова языка Java, которые вы можете использовать для изменения элементов (классов, методов, переменных). В Java есть две категории модификаторов: модификаторы доступа и другие модификаторы.
Строка, в которой объявляется метод, со всеми модификаторами, будет влиять на то, как этот метод будет вызываться из других классов и методов. Результат какого типа он будет возвращать и какие возможны ошибки при его работе.
Примеры модификаторов доступа
В Java есть четыре модификатора доступа:
* public - открытый элемент. Доступ к нему можно получить из класса, снаружи класса, внутри и снаружи пакета, в пределах одного проекта (Java файл может иметь только один public класс и множество подклассов). Классы могут быть только public и default;
* private — элемент доступен только внутри объявленного им класса;
* default — элементу с модификатором по умолчанию, есть доступ только внутри пакета. Устанавливается ко всем полям и методам, в которых не подставлены другие модификаторы;
* protected — можно получить доступ внутри и снаружи пакета через дочерний класс и только в классе наследника (дочерний). Применяется в основном только к методам и давольно редко, нужно основание.;

Для чего же следует использовать модификаторы доступа? При разработке ООП-кода классы проектируются с целью моделирования сущностей предметной области. Описывая класс разработчик выделяет внешнюю и внутреннюю реализации типа. Модификаторы доступа позволяют при проектировании класса указать что есть интерфейс (внешняя реализация) и открыть ее для использования, а что есть внутренняя реализация.  
Примеры других модификаторов (для классов, полей, методов, интерфейсов и так далее) — семь:
* static - переменные с данным модификатором относятся не к объекту, а к классу и делится на все объекты класса. Если мы создаем одну переменную на все объекты (допустим сделать счетчик на все созданные объекты). Static метод может быть только один на класс. 1. Нужен в том случае, если методу не нужно знать переменные объекта (класс Math). 2.Статические методы не наследуются и, следовательно, не переопределяются. 3. Если метод не может существовать без данного объекта, тогда этот метод не может быть статическим. 4. Если метод статический, то он не имеет доступа к нестатическим полям (можно тогда передать в него переменные через параметры). Если нужно наоборот, нестатические к статическим, без проболем  5. Если в классе только статический метод, тогда лучше делать класс final. 6. Статические методы являются методами класса, а не объекта. 6. У статических методов нет ссылки this. 7. Можно создавать статический блок данных и их может быть несколько в классе; 8. При запуске программы статические поля инициализируются сразу по умолчанию, а потом уже сверху вниз в порядке инициализации в классе.
* final - переменного данного типа не может быть изменена, то есть константа (пишутся все заглавными символами). Константы делаем static, что бы принадлежала ко всем объектам, при необходимости. Методы с модификатором final не могут быть переопределены в дочерних классах. Ключевое слово final запрещает изменение метода.
